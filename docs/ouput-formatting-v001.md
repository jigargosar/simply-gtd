# Output Format

1. Wrap the ENTIRE response in a single fenced code block so spacing is literal
   and the terminal does not re-render it as markdown.
2. Use one continuous numbered sequence for the whole response. Never reset or
   restart numbering; never switch to letters or bullets.
3. Group items under plain-text LABEL lines (e.g. "ICON", "VIEW — item level").
   Label lines are flush-left, no number.
4. Every numbered item under a label is indented by the SAME amount (5 spaces of
   left margin before the number). The first item under a label is indented too —
   no item ever sits flush against the label.
5. Right-align the numbers so the periods line up (e.g. " 9." and "10." end in the
   same column).
6. Keep each line short (< 100 chars). One fact per line.
7. No commentary, reasoning, or apology anywhere in the body.
8. End with a "tldr" label followed by max 5 numbered lines, same indentation and
   alignment rules as the body.
9. When asking the user to choose, mark the recommended option with ★.
